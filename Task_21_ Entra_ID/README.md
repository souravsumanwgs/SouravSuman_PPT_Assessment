# Task 21 – Entra ID (Users, Roles, Basic Identity Management)

## 📌 Objective
Explore **users and role management** in **Microsoft Entra ID** (Azure Active Directory).  
This task demonstrates basic **identity and access management** in Azure.

---

## 🛠️ Services Used
- Microsoft Azure Portal  
- Microsoft Entra ID (Azure Active Directory)  

---

## 🌐 Implementation Steps

### Step 1: Login to Azure Portal
- Access the **Azure Portal** using your credentials.

### Step 2: Open Microsoft Entra ID
- Search for **Microsoft Entra ID** in the portal search bar.  
- Open the **Entra ID Dashboard**.

### Step 3: Create a New User
1. Click on **Users** from the left-hand menu.  
2. Click **New User → Create New User**.  
3. Enter the required details:  
   - **Name**: *[User Full Name]*  
   - **Username**: *[User Principal Name]*  
   - Basic details (password, profile info, etc.)  
4. Click **Create** to add the user.  

### Step 4: Assign a Role
1. Go to **Roles and Administrators** from the Entra ID menu.  
2. Select a **basic role** (e.g., **Reader**).  
3. Assign the role to the newly created user.  

### Step 5: Verify User and Role
- Confirm that the user appears in the **Users list**.  
- Ensure the **role assignment** is correctly reflected for the user.

## 🔍 Key Learnings
- Entra ID is the **identity management service** for Azure.  
- Users can be created and assigned roles to control **access permissions**.  
- Basic roles like **Reader** limit users to **view-only access**.  
- Understanding user and role management is fundamental for **cloud security**.
